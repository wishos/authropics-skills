# 前端设计 (Frontend Design) 指南

## 触发条件

当用户要求构建 Web 组件、页面、artifacts、海报或应用程序（示例包括网站、登陆页面、仪表板、React 组件、HTML/CSS 布局，或在美化任何 Web UI）时使用此 Skill。生成避免通用 AI 美学的创意、精细的代码和 UI 设计。

---

## 设计思维

在编码之前，了解上下文并承诺大胆的美学方向：
- **目的**：这个界面解决什么问题？谁使用它？
- **语气**：选择一个极端：残酷极简、极繁主义、复古未来主义、有机/自然、奢华/精致、活泼/玩具、编辑/杂志、粗野主义/原始、装饰艺术/几何、柔软/粉彩、工业/实用等。有很多选择可以用来激发灵感，但设计一个真正符合美学方向的。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：什么会让这个令人难忘？人们会记住的一件事是什么？

**关键**：选择清晰的概念方向并精确执行。精细极繁主义和精炼极简主义都有效——关键是有意性，而不是强度。

然后实现工作的代码（HTML/CSS/JS、React、Vue 等）：
- 生产级且功能
- 视觉震撼且令人难忘
- 具有清晰美学观点的内聚
- 在每个细节上精炼

---

## 前端美学指南

### 字体排版

选择美丽、独特、有趣的字体。避免通用字体如 Arial 和 Inter；相反，选择提升前端美学的独特选择意外、有特色的字体选择。将独特的展示字体与精炼的正文字体配对。

### 颜色和主题

承诺内聚美学。使用 CSS 变量保持一致。主导颜色与锐利强调色优于胆怯、均匀分布的调色板。

### 动画

使用动画效果和微交互。优先考虑 HTML 的纯 CSS 解决方案。使用 Motion  library for React（如果有）。专注于高影响时刻：一个精心编排的页面加载与交错揭示（animation-delay）比零散的微交互创造更多 delight。使用滚动触发和悬停状态带来惊喜。

### 空间构图

意外的布局。不对称。重叠。对角线流动。打破网格的元素。慷慨的负空间或受控密度。

### 背景和视觉细节

创造氛围和深度，而不是默认纯色。添加与整体美学匹配的上下文效果和纹理。应用创意形式，如渐变网格、噪声纹理、几何图案、分层透明度、戏剧性阴影、装饰边框、自定义光标和颗粒叠加。

### 永远不要使用通用 AI 生成美学

如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、陈腐的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏上下文特定特征的 cookie-cutter 设计。

创造性地解释并做出感觉真正为上下文设计的意外选择。没有设计应该是一样的。在不同光线和深色主题、不同字体、不同美学之间变化。永远不要跨代收敛于常见选择（例如 Space Grotesk）。

**重要**：将实现复杂性匹配到美学愿景。极繁设计需要复杂的代码与广泛的动画和效果。精炼或精炼设计需要克制、精确，仔细关注间距、字体排版和微妙细节。优雅来自于很好地执行愿景。
