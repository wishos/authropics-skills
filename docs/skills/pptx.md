# PPTX (PowerPoint) 处理指南

## 触发条件

当涉及 .pptx 文件的任何方式时使用此 Skill——作为输入、输出或两者兼有。包括：
- 创建幻灯片组、演示文稿或演示文稿

- 合并或拆分幻灯片文件
- 使用模板、布局、演讲者备注或评论

**触发条件**：用户提及"deck"、"slides"、"presentation"，或引用 .pptx 文件名。

## 设计建议

**不要创建无聊的幻灯片。** 白色背景上的纯要点不会给任何人留下深刻印象。考虑此列表中的想法用于每张幻灯片。

### 开始之前

- **选择大胆、符合内容的配色方案**：配色方案应该为这个主题专门设计。如果将您的颜色完全换到不同的演示文稿中仍然可以"工作"，则您没有做出足够具体的选择。
- **优势优于平等**：一种颜色应该占主导地位（60-70% 视觉权重），有 1-2 种支持色调和一种锐利的强调色。永远不要给所有颜色相同的权重。
- **深浅对比**：深色背景用于标题 + 结论幻灯片，浅色用于内容（"三明治"结构）。或者全程使用深色以获得高级感。
- **坚持视觉主题**：选择一个独特元素并重复它——圆角图像框架、彩色圆圈中的图标、粗单边边框。将其贯穿每张幻灯片。

### 配色方案

选择匹配您主题的颜色——不要默认使用通用蓝色。以这些配色方案为灵感：

|------|------|------|--------|
| **Midnight Executive** | `1E2761` (海军蓝) | `CADCFC` (冰蓝) | `FFFFFF` (白色) |
| **Forest & Moss** | `2C5F2D` (森林) | `97BC62` (苔藓) | `F5F5F5` (奶油色) |
| **Coral Energy** | `F96167` (珊瑚) | `F9E795` (金色) | `2F3C7E` (海军蓝) |

| **Ocean Gradient** | `065A82` (深蓝) | `1C7293` (青色) | `21295C` (午夜) |
| **Charcoal Minimal** | `36454F` (炭灰) | `F2F2F2` (米白) | `212121` (黑色) |

### 每张幻灯片

**每张幻灯**——图像、图表、图标或片都需要视觉元素形状。仅文本幻灯片容易被遗忘。

**布局选项：**
- 双列（文本在左，插图在右）
- 图标 + 文本行（图标在彩色圆圈中，粗体标题，下方描述）
- 2x2 或 2x3 网格（图像在一侧，内容块网格在另一侧）
- 半出血图像（完整左或右侧）带内容叠加

**数据显示：**
- 大数字强调（大数字 60-72pt，下方小标签）
- 对比列（之前/之后、利弊、并列选项）
- 时间线或流程（编号步骤、箭头）

**视觉打磨：**
- 彩色小圆圈中的图标靠近章节标题
- 斜体强调文本用于关键统计数据或标语

### 字体排版

**选择有趣的字体配对**——不要默认使用 Arial。选择有特色的标题字体并与干净的正文字体配对。

| 标题字体 | 正文字体 |
|---------|---------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 尺寸 |
|------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文文本 | 14-16pt |
| 说明文字 | 10-12pt 静音 |

### 间距

- 最小 0.5" 边距
- 内容块之间 0.3-0.5"
- 留有呼吸空间——不要填满每一寸

### 避免（常见错误）

- **不要重复相同的布局**——在幻灯片之间变化列、卡片和强调
- **不要居中正文文本**——左对齐段落和列表；仅居中标题
- **不要吝啬尺寸对比**——标题需要 36pt+ 才能从 14-16pt 正文中脱颖而出
- **不要默认使用蓝色**——选择反映特定主题的颜色
- **不要随机混合间距**——选择 0.3" 或 0.5" 间隙并一致使用
- **不要美化一张幻灯片而让其余的保持朴素**——要么完全投入，要么全程保持简单
- **不要创建仅文本幻灯片**——添加图像、图标、图表或视觉元素；避免纯标题 + 要点
- **不要忘记文本框内边距**——当将线条或形状与文本边缘对齐时，在文本框上设置 `margin: 0` 或偏移形状以考虑内边距
- **不要使用低对比度元素**——图标和文本都需要与背景形成强烈对比；避免浅色背景上的浅文本或深色背景上的深文本
- **永远不要在标题下使用强调线**——这些是 AI 生成幻灯片的标志；改用空白或背景颜色

## QA（必需）

**假设存在问题。您的任务是找到它们。**

您的第一次渲染几乎不正确。将 QA 视为寻找错误，而不是确认步骤。如果在第一次检查中没发现问题，说明您看得不够仔细。

### 内容 QA

```bash
python -m markitdown output.pptx
```

检查缺失内容、拼写错误、错误顺序。

**使用模板时，检查剩余占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回结果，在声明成功之前修复它们。

### 视觉 QA

**⚠️ 使用子代理**——即使只有 2-3 张幻灯片。您一直盯着代码，会看到您期望的，而不是实际存在的。子代理有新鲜的眼睛。

将幻灯片转换为图像（参见下面的"转换为图像"），然后使用此提示：

```
视觉检查这些幻灯片。假设存在问题——找到它们。

查找：
- 重叠元素（形状中的文本、词语中的线条、堆叠元素）
- 边缘/框边界处的文本溢出或截断
- 为单行文本布置的装饰线条但标题换行两行
- 与上方内容冲突的来源引文或页脚
- 元素太近（< 0.3" 间隙）或卡片/部分几乎接触
- 不均匀间隙（一处大空白，另一处拥挤）
- 距幻灯片边缘的边距不足（< 0.5"）
- 列或类似元素不一致对齐
- 低对比度文本（例如米色背景上的浅灰色文本）
- 低对比度图标（例如深色背景上的深色图标，没有对比色圆圈）
- 文本框太窄导致过度换行
- 剩余占位符内容

对于每张幻灯片，列出问题或关注区域，即使是小问题。

读取并分析这些图像：
1. /path/to/slide-01.jpg (预期: [简要描述])
2. /path/to/slide-02.jpg (预期: [简要描述])

报告发现的所有问题，包括小问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图像 → 检查
2. **列出发现的问题**（如果没有发现，更仔细地再看一次）
3. 修复问题
4. **重新验证受影响的幻灯片**——一次修复通常会产生另一个问题
5. 重复直到完整检查没有发现新问题

**在完成至少一个修复-验证循环之前不要声明成功。**

## 转换为图像

将演示文稿转换为单独的幻灯片图像以进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这将创建 `slide-01.jpg`、`slide-02.jpg` 等。

要在修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
