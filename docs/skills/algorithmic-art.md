# 算法艺术 (Algorithmic Art) 指南

## 触发条件

当用户请求使用代码创建艺术、生成艺术、算法艺术、流场或粒子系统时使用此 Skill。创建原创的算法艺术，而不是复制现有艺术家的作品以避免版权侵权。

---

## 概述

算法哲学是通过代码表达的计算机美学运动。输出 .md 文件（哲学）、.html 文件（交互式查看器）和 .js 文件（生成算法）。

这个过程分两步进行：
1. 算法哲学创建（.md 文件）
2. 通过创建 p5.js 生成艺术来表达（.html + .js 文件）

---

## 第一步：算法哲学创建

首先，创建一个**算法哲学**（不是静态图像或模板），将通过以下方式解释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪声场、有机系统
- 粒子、流、场、力
- 参数变化和受控混沌

### 关键理解

- **收到的**：用户的一些微妙输入或说明，应作为基础加以考虑，但不应限制创作自由
- **创建的**：算法哲学/生成美学运动
- **接下来发生什么**：同一版本接收哲学并**用代码表达它** - 创建 p5.js 草图，90% 是算法生成，10% 是基本参数

### 如何生成算法哲学

**命名运动**（1-2 个词）："Organic Turbulence" / "Quantum Harmonics" / "Emergent Stillness"

**阐述哲学**（4-6 段 - 简洁但完整）：

为了捕捉**算法本质**，表达这种哲学如何通过以下方式体现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面只应提及一次。除非添加新深度，否则避免重复关于噪声理论、粒子动力学或数学原理的观点。
- **反复强调工艺**：哲学必须多次强调最终算法应该看起来像是花费了无数小时开发的，经过精心打磨的，是领域顶尖专家的产物。这种框架至关重要——重复"精心制作的算法"、"深层计算专业知识的产品"、" painstaking optimization"、"master-level implementation"等短语。
- **留出创作空间**：对算法方向要具体，但足够简洁，以便下一个 Claude 也有空间在极高的工艺水平上做出解释性实现选择。

哲学必须引导下一个版本**用算法方式**表达想法，而不是静态图像。之美存在于过程中，而不是最终帧。

### 哲学示例

**"Organic Turbulence"**
哲学：混沌受自然法则约束，秩序从无序中涌现。
算法表达：由分层 Perlin 噪声驱动的流场。数千个粒子遵循向量力，它们的轨迹积累成有机密度图。多层噪声产生湍流区域和安静区域。颜色从速度和密度中出现——快速粒子燃烧明亮，缓慢粒子消逝到阴影。算法运行直到平衡——经过无数次迭代精心调参的平衡，每一个参数都由计算美学大师精炼。

**"Quantum Harmonics"**
哲学：离散实体表现出波浪干扰模式。
算法表达：在网格上初始化的粒子，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干扰——建设性干扰产生明亮节点，破坏性干扰产生空洞。简单谐波运动产生复杂的涌现曼陀罗。这是经过精心频率校准的结果，每个比率都经过精心选择以产生共振之美。

**"Recursive Whispers"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分分支结构。每个分支稍微随机化但受黄金比例约束。L 系统或递归细分生成树状形式，感觉既是数学的又是有机的。微妙的噪声扰动打破完美对称。线条粗细随着每个递归级别递减。每个分支角度都是深度数学探索的产物。

**"Field Dynamics"**
哲学：不可见的力量通过其对物质的影响变得可见。
算法表达：由数学函数或噪声构建的向量场。粒子在边缘生成，沿场线流动，当达到平衡或边界时消失。多个场可以吸引、排斥或旋转粒子。可视化只显示轨迹——幽灵般不可见力量的证据。计算舞蹈通过力平衡精心编排。

**"Stochastic Crystallization"**
哲学：随机过程结晶成有序结构。
算法表达：随机圆填充或 Voronoi 镶嵌。从随机点开始，通过松弛算法演化。单元格相互推挤直到平衡。颜色基于单元格大小、邻居数量或距中心距离。涌现的有机体镶嵌感觉既随机又不可避免。每个种子产生独特的结晶之美——生成算法大师级标记。

### 基本原则

- **算法哲学**：创建通过代码表达的计算机世界观
- **过程优于产物**：始终强调之美从算法执行中涌现——每次运行都是独特的
- **参数表达**：想法通过数学关系、力量、行为传达——不是静态构图
- **创作自由**：下一个 Claude 用算法方式解释哲学——提供创作实现空间
- **纯生成艺术**：这是关于制作**活算法**，而不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，经过无数次迭代精炼，是计算美学领域绝对顶尖专家的深度专业知识的产品

## 第二步：推断概念种子

**关键步骤**：在实现算法之前，从原始请求中识别微妙的概念线索。

**基本原则**：
主题是一个**微妙、精炼的引用嵌入算法本身**——不总是字面意思，总是复杂的。熟悉该主题的人应该直观地感受到它，而其他人只是体验到精妙的算法构图。算法哲学提供计算语言。推断的概念提供灵魂——安静的概念 DNA 不可见地编织到参数、行为和涌现模式中。

**非常重要**：引用必须精炼到增强作品深度而不自我宣布。就像爵士音乐家引用另一首歌——只有知道的人会注意到，但每个人都欣赏生成之美。

## 第三步：p5.js 实现

哲学和概念框架确立后，通过代码表达。暂停收集想法，然后按照以下说明进行。

### ⚠️ 第 0 步：首先阅读模板 ⚠️

**关键：在编写任何 HTML 之前：**

1. **阅读** `templates/viewer.html` 使用 Read 工具
2. **研究** 确切的结构、样式和 Anthropic 品牌
3. **使用该文件作为字面起点**——不仅仅是灵感
4. **保持所有固定部分完全按照显示的**（标题、侧边栏结构、Anthropic 颜色/字体、种子控件、操作按钮）
5. **只替换** 文件注释中标记的可变部分（算法、参数、参数的 UI 控件）

**避免：**
- ❌ 从头创建 HTML
- ❌ 发明自定义样式或配色方案
- ❌ 使用系统字体或深色主题
- ❌ 更改侧边栏结构

**遵循这些做法：**
- ✅ 复制模板的确切 HTML 结构
- ✅ 保持 Anthropic 品牌（Poppins/Lora 字体、浅色、渐变背景）
- ✅ 保持侧边栏布局（Seed → Parameters → Colors? → Actions）
- ✅ 只替换 p5.js 算法和参数控件

模板是基础。在它上面构建，而不是重建它。

---

### 技术要求

**种子随机性（Art Blocks 模式）：**
```javascript
// 始终使用种子以确保可重复性
let seed = 12345; // 或从用户输入哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循哲学：**

为了建立从算法哲学自然涌现的参数，考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以确保可重复性
  // 颜色
  // 添加控制你的算法的参数：
  // - 数量（多少？）
  // - 比例（多大？多快？）
  // - 概率（多可能？）
  // - 比率（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**设计有效参数，专注于系统需要可调的属性，而不是考虑"模式类型"。**

**核心算法 - 表达哲学：**

**关键**：算法哲学应该决定要构建什么。

通过代码表达哲学，避免思考"我应该使用哪种模式？"而是思考"如何通过代码表达这种哲学？"

如果哲学是关于**有机涌现**，考虑使用：
- 随着时间积累或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学是关于**数学之美**，考虑使用：
- 几何关系和比率
- 三角函数和谐波
- 精确计算产生意外模式

如果哲学是关于**受控混沌**，考虑使用：
- 严格边界内的随机变化
- 分叉和相变
- 从无序中涌现秩序

算法从哲学流出，而不是从选项菜单流出。

让概念本质指导创作和原创选择。为这个特定请求构建表达愿景的东西。

**画布设置**：标准 p5.js 结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系统
}

function draw() {
  // 你的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：为了达到大师级水平，创建感觉像是经过生成艺术大师无数次迭代精炼的算法。仔细调整每个参数。确保每个模式都是有目的地涌现。这不是随机噪声——这是通过深度专业知识精炼的**受控混沌**。

- **平衡**：复杂性没有视觉噪声，秩序没有僵化
- **色彩和谐**：深思熟虑的调色板，而不是随机 RGB 值
- **构图**：即使在随机中，保持视觉层次和流动
- **性能**：平滑执行，如果动画则优化实时性能
- **可重复性**：相同的种子总是产生相同的输出

### 输出格式

输出：
1. **算法哲学** - 作为 markdown 或文本解释生成美学
2. **单个 HTML 产物** - 来自 `templates/viewer.html` 的自包含交互式生成艺术（参见第 0 步和下一节）

HTML 产物包含一切：p5.js（来自 CDN）、算法、参数控件和 UI——一个可以在 claude.ai artifacts 或任何浏览器中立即运行的文件。从模板文件开始，而不是从零开始。

## 交互式产物创建

**提醒：`templates/viewer.html` 应该已经被阅读（参见第 0 步）。使用该文件作为起点。**

允许探索生成艺术，创建单一、自包含的 HTML 产物。确保此产物在 claude.ai 或任何浏览器中立即工作——无需设置。所有内容内联。

### 关键：什么是固定 vs 可变

`templates/viewer.html` 文件是基础。它包含所需的确切结构和样式。

**固定（始终按显示的精确包含）：**
- 布局结构（标题、侧边栏、主画布区域）
- Anthropic 品牌（UI 颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 转到按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个艺术品自定义）：**
- 整个 p5.js 算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 一些艺术需要颜色选择器
  - 一些艺术可能使用固定颜色
  - 一些艺术可能是单色（不需要颜色控件）
  - 根据艺术需要决定

**每个艺术品应该有独特的参数和算法！** 固定部分提供一致的 UX——其他一切表达独特愿景。

### 必需功能

**1. 参数控件**
- 数字参数的滑块（粒子数量、噪声比例、速度等）
- 调色板的颜色选择器
- 参数更改时实时更新
- 恢复默认值的重置按钮

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮获取随机种子
- 跳转到特定种子的输入字段
- 请求时生成 100 个变体（种子 1-100）

**3. 单一产物结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js 来自 CDN - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 简洁、最小 */
    /* 画布在顶部，控件在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有 p5.js 代码内联在这里
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI 处理程序
    // 一切自包含
  </script>
</body>
</html>
```

**关键**：这是一个单一产物。没有外部文件，没有导入（除了 p5.js CDN）。一切内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终按显示精确包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术创建控件：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
根据参数数量添加尽可能多的 control-group div。

**3. 颜色（可选/可变）** - 如果艺术需要可调整颜色则包含：
- 如果用户应该控制调色板则添加颜色选择器
- 如果艺术使用固定颜色则跳过此部分
- 如果艺术是单色则跳过
- 根据艺术需要决定

**4. 操作（固定）** - 始终按显示精确包含：
- 重新生成按钮
- 重置按钮
- 下载 PNG 按钮

**要求**：
- 种子控件必须工作（上/下/随机/跳转/显示）
- 所有参数必须有 UI 控件
- 重新生成、重置、下载按钮必须工作
- 保持 Anthropic 品牌（UI 样式，不是艺术颜色）

### 使用产物

HTML 产物立即工作：
1. **在 claude.ai 中**：显示为交互式 artifact - 即时运行
2. **作为文件**：保存并在任何浏览器中打开 - 无需服务器
3. **分享**：发送 HTML 文件——它完全自包含

## 变体与探索

产物默认包含种子导航（上/下/随机按钮），允许用户探索变体而无需创建多个文件。如果用户想要突出显示特定变体：

- 包含种子预设（按钮如"变体 1：种子 42"、"变体 2：种子 127"等）
- 添加"画廊模式"并排显示多个种子的缩略图
- 全部在同一个单一产物内

这就像从同一印版创建一系列印刷品——算法一致，但每个种子揭示其潜力的不同方面。交互性质意味着用户通过探索种子空间发现他们自己的最爱。

## 创意过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。这个过程涉及：

1. **解释用户的意图** - 正在寻求什么美学？
2. **创建算法哲学**（4-6 段）描述计算方法
3. **用代码实现** - 构建表达这种哲学的算法
4. **设计适当的参数** - 什么应该是可调的？
5. **构建匹配的 UI 控件** - 那些参数的滑块/输入

**常量**：
- Anthropic 品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含 HTML 产物

**其他都是可变的**：
- 算法本身
- 参数
- UI 控件
- 视觉结果

为获得最佳结果，相信创意并让哲学指导实现。
